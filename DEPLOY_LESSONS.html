nginx เลือกเว็บยังไง (สิ่งที่เคยเข้าใจผิด)

  • nginx ดู ชื่อโดเมน (Host header) ที่เข้ามา แล้วจับคู่กับ server_name
  • salesbot.ohmai.me → ตรง block ของ SalesBot
  • โดเมนอื่น ที่ไม่ตรงใคร → ไปที่ default_server (ต้องเป็น Hermes)
กับดัก: server_name _; ไม่ใช่ catch-all จริง! ถ้าโดเมนไม่ตรงใคร nginx จะส่งไป server block แรกที่ listen 443 — ต้องกำหนด default_server ให้ Hermes ชัดเจน ไม่งั้นจะหลงไป SalesBot

4กฎเหล็ก (จำให้ขึ้นใจ)

✅ ทำแบบนี้

  • Push ก่อน เสมอ แล้วค่อย pull บน VPS
  • แก้ app.html/nginx.conf → force-recreate
  • git status ก่อน pull ทุกครั้ง
  • verify ทั้ง 2 เว็บ หลัง deploy
  • config สำคัญที่อยู่บน VPS → ย้ายเข้า git ให้หมด

❌ ห้ามทำ

  • git stash โดยไม่ดูว่าทิ้งอะไร
  • แก้ไฟล์ตรงๆ บน VPS แล้วไม่ commit
  • คิดว่า nginx -s reload พอ (bind-mount ต้อง recreate)
  • เช็คแค่เว็บเดียวแล้วคิดว่าจบ
  • ตั้ง ssl_session_cache ชื่อ zone ซ้ำกันคนละ size
เรื่อง bind-mount + git pull (จุดพลาดบ่อย): ไฟล์อย่าง app.html, nginx.conf ถูก mount แบบ "ไฟล์เดี่ยว" เข้า container เวลา git pull เขียนไฟล์ใหม่ → inode เปลี่ยน แต่ container ยังชี้ไฟล์เก่า → nginx -s reload ไม่เห็นของใหม่ ต้อง --force-recreate เท่านั้น

5เหตุการณ์จริง 14 มิ.ย. 2026 — ทำ SalesBot ล่มได้ยังไง

ตั้งใจ deploy แค่ Hermes แต่ทำ SalesBot ล่ม เพราะลืมว่า nginx ตัวเดียวเสิร์ฟ 2 เว็บ

① git stash ทิ้ง config ที่ใช้งานจริง
ตอน pull เจอ nginx.conf ค้าง เลย stash ทิ้ง — แต่ในนั้นมี include salesbot.conf ที่ทำให้ SalesBot ทำงาน → SalesBot หาย route → HTTP 526
② กู้ include กลับ → nginx crash ทั้งคู่
salesbot.conf ใช้ ssl_session_cache shared:SSL:50m แต่ Hermes ใช้ชื่อ zone SSL เท่ากันแต่ 10m → ชื่อ zone ซ้ำ size ขัดกัน → nginx crash loop → ดับทั้ง 2 เว็บ
③ แก้ชื่อ zone → risacore เด้งไป SalesBot
เปลี่ยนชื่อ zone เป็น salesbot_ssl nginx ขึ้นได้ แต่ salesbot block ถูก include ก่อน Hermes → กลายเป็น default_server → risacore.ohmai.me แสดงหน้า SalesBot
④ ตั้ง Hermes เป็น default_server ชัดเจน → จบ ✅
เพิ่ม listen 443 ssl http2 default_server; + server_name risacore.ohmai.me _; ใน Hermes block → ทั้ง 2 เว็บกลับมาปกติ
