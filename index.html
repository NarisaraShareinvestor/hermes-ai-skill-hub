n8n Workflows ที่ต้องมี

n8n ทำหน้าที่เป็น automation layer คอยเชื่อมระบบต่าง ๆ เข้าด้วยกัน

Workflow Trigger Action Output
Skill Review Workflow มี skill ใหม่ถูก submit ส่งรายละเอียดไป Telegram พร้อมปุ่ม Approve/Reject อัปเดตสถานะ skill

Admin Approval Workflow มี action เสี่ยง เช่น publish, delete, external share ขออนุมัติจาก admin ทำ action หลัง approve เท่านั้น

Hostinger Deployment Design

ออกแบบให้รันบน Hostinger VPS ด้วย Docker เพื่อเริ่มง่ายและขยายต่อได้

Container ที่ควรมี

hostinger-vps ├── nginx-proxy │ └── HTTPS / domain routing │ ├── ai-portal │ └── Frontend: Chat UI + Skill Store │ ├── backend-api │ └── FastAPI / Node.js API │ ├── ai-agent │ └── Hermes / OpenAI / Local LLM connector │ ├── n8n │ └── Workflow automation │ ├── postgres │ └── users, teams, skills, logs │ ├── vector-db │ └── document embeddings │ └── redis └── queue / cache / background jobs

Domain ที่แนะนำ

ai.company.com AI Portal
api.company.com Backend API
n8n.company.com n8n Admin
agent.company.com AI Agent Internal API
Production จริงควรปิด n8n admin ด้วย login, VPN หรือ IP allowlist และอย่า expose internal agent API แบบ public

API Endpoints ที่ควรมี

Backend API เป็นตัวกลางระหว่าง Portal, AI Agent, n8n และ Database

Endpoint Method หน้าที่
/api/chat POST คุยกับ Personal AI Assistant และให้ agent เลือก skill ที่เหมาะ
/api/skills GET / POST ดู skill ทั้งหมดหรือสร้าง skill ใหม่
/api/skills/{id}/run POST เรียกใช้ skill ตาม input ที่ส่งมา
/api/skills/{id}/submit-review POST ส่ง skill เข้า review และ trigger n8n workflow
/api/reviews/{id}/approve POST อนุมัติ skill หลังจากกดจาก Telegram หรือ Admin Portal
/api/documents/index POST ส่งเอกสารเข้า indexing pipeline

RBAC Department Permission

Approve Reject

Audit Log

Logs Traceability

Data Boundary

Safe Tools
